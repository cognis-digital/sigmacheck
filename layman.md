Sigmacheck is a tool for checking that security detection rules are written correctly. Security teams write "Sigma rules" — small text files that describe suspicious behavior to look for in system logs — and sigmacheck reads those files, flags mistakes, and then runs the rule against sample events to confirm it actually catches the bad activity it was designed to detect. It works entirely from the command line with no account or internet connection required, and outputs results as plain text or JSON so it fits into automated testing pipelines. It is aimed at security engineers and detection-engineering teams who write or review Sigma rules and want a fast, self-contained way to validate them before deploying to a SIEM.
